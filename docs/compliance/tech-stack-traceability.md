This matrix expands every `MUST`, `MUST NOT`, and `SHOULD` in `TECH_STACK.md`. It also gives each
unkeyworded normative runtime, package, technology, version, image, command, and deferred-choice
contract an independently statusable row. RFC 2119 boilerplate, examples, rationale, optional `MAY`
behavior without a mandatory constraint, and non-normative references are excluded.

Status meanings and workstream labels follow `spec-traceability.md`. An **Implemented** row names a
direct deterministic test or command. Presence-only evidence is **Partial**, and requirements that
need a supported host, registry, provider, or GitHub run remain **External proof**.
