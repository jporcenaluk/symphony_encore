# SPEC.md traceability matrix

This matrix covers every `MUST`, `MUST NOT`, and `SHOULD` in `SPEC.md`. It also expands lists
introduced by normative phrases such as “MUST provide,” “MUST ship,” and “Required operations.” It
excludes RFC 2119 boilerplate, examples, rationale, Appendix A change notes, and optional `MAY`
behavior unless a mandatory constraint governs that behavior.

Status meanings:

- **Implemented**: the repository contains the behavior and a direct deterministic test.
- **Partial**: code or a narrow test exists, but some named behavior or proof is absent.
- **Missing**: no implementation and direct evidence were found.
- **External proof**: repository code may exist, but completion requires live provider, deployment,
  or operator evidence that has not been recorded.

Workstreams: **W0** evidence/conformance; **W1** runtime foundations; **W2** durability/recovery;
**W3** configuration/control mutation; **W4** agent/security boundary; **W5** scheduler; **W6**
lifecycle/budget/review/merge; **W7** learning/observability/retention; **W8** Control API; **W9** web
UI; **W10** delivery/governance; **W11** real integration.
