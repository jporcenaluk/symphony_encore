# CICD.md traceability matrix

This matrix expands the delivery contract in `CICD.md` in source order. It includes each explicit or
inherited `MUST`, `MUST NOT`, and `SHOULD`, plus unkeyworded contracts that define required workflow,
artifact, retention, rollback, command, or adoption behavior. RFC boilerplate, `MAY` options, and
non-normative references are excluded. Authored configuration is not remote proof. Status and
workstream labels use the definitions in `README.md`.

A normative lead is not duplicated as an umbrella row when its child list completely instantiates
the contract. In particular, line 274's update-policy lead is represented by its independently
statusable child rows.
